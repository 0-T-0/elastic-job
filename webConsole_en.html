Elastic-job

Elastic-Job is a distributed scheduled job framework, based on Quartz and Curator.

View on GitHub Download .zip Download .tar.gz

Web console

Web Console for elastic-job provides as a war file, user can deploy it into a servlet container such as tomcat or jetty.

You can get elastic-job-console.war from Maven Central Repository or compile from source code.

  • Login

    Default username and password is root/root, user can modify them from conf\auth.properties.

  • Feature list

    Security login

    Registry center management

    Jobs dimension display

    Servers dimension display

    Modify job configuration

    Job pause and resume

  • Design idea

    Web console and elastic-job is totally separate, the only relationship is registry center. Web console load from or update configuration to registry center.

    Web console only can control job is running or not, but cannot control job process start or kill. Because web console and job server is distributed, they cannot control each other.

  • Limitations

    Add job. Because of job can add by first run automatically, use web console to add job is unnecessary.

    Kill job. Even remove job info from Zookeeper, the job still run, and maybe cause some error occur. If you want kill job, must go to job server and kill the process.

    Delete job server. Because remove server node has big risk, so don't support right now.
