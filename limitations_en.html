Elastic-job

Elastic-Job is a distributed scheduled job framework, based on Quartz and Curator.

View on GitHub Download .zip Download .tar.gz

Limitations

  • Cannot modify job name after job started, if modify job name will be recognized as a new job.
  • Same job server only can run one same job instance, because job will register to zookeeper by server IP.
  • If job servers crashed, add new one or modify sharding total count, will trigger re-sharding. Trigger re-sharding will cause running data-flowed jobs finish current trun and then stop, resume running after re-sharding.
  • Only enable "monitorExecution" can make sure job idempotency, but will affect performance, recommend disable "monitorExecution" for instantaneous jobs (for example: trigger per 5 seconds) and implement idempotency by yourself.
  • Elastic-Job cannot remove job servers automatically, because it cannot judge server is crashed or remove normally. So if you want remove job servers, please remove znode from zookeeper manually.
